# Product Brief: MindReminder

**Date:** 2025-12-13
**Author:** Administrator

---

## Executive Summary

**MindReminder** 是一款面向自主学习者的桌面应用，专注于帮助用户通过科学的间隔重复方法巩固知识、养成高效复习习惯。与现有工具不同，MindReminder 提供高度可定制的复习算法、直观的日历式界面，并将知识管理、日记记录和提醒事项整合为统一的个人成长中枢。产品采用本地数据存储，为用户提供完全的数据掌控权和隐私保护。

目标用户是正在准备考试、面试或持续学习的知识工作者，他们主要在电脑前学习，需要一个既科学又灵活的复习管理系统。

产品将分阶段演进：从桌面端本地存储起步（MVP），逐步支持云端同步和移动端应用，最终形成跨平台的个人成长生态系统。

## Core Vision

### Problem Statement

知识学习者面临"学了就忘"的普遍痛点。尽管科学研究证明间隔重复能有效对抗遗忘曲线，但：

- **缺乏有效工具**：市面上缺少真正适合个人学习节奏的桌面复习工具
- **灵活性不足**：现有工具（如Anki、SuperMemo）复习频率固定，无法根据个人情况调整，导致复习量过大或不足
- **信息孤岛**：知识点、日常记录、待办事项分散在不同工具中，无法形成完整的学习画像
- **可视化欠佳**：难以直观了解学习进度、复习计划和记忆进展

### Problem Impact

对于正在准备考试、面试或技能提升的学习者而言：

- **时间浪费**：没有科学复习导致学习效率低下，重复学习已掌握的内容或遗忘关键知识
- **焦虑感**：不知道自己还需要复习什么、何时能真正掌握，缺乏学习掌控感
- **动力不足**：看不到学习进展和成果，难以坚持复习计划
- **效果不佳**：最终在考试或面试中因知识遗忘而影响表现

### Why Existing Solutions Fall Short

现有间隔重复工具存在明显缺陷：

**灵活性问题：**

- 复习间隔算法固化，无法根据个人学习节奏和复习能力调整
- 无法针对不同难度的知识点设置不同的复习策略
- 缺少"复习频率系数"等个性化参数

**用户体验问题：**

- 多数工具以移动端或Web端为主，桌面体验不佳
- 界面复杂或过于简陋，无法清晰展示学习全貌
- 缺少日历式可视化，难以快速了解过去记录和未来计划

**功能局限：**

- 仅关注知识点记忆，无法承载日记、待办等完整学习生活
- 缺少对"何时真正记住"的预测和进度追踪
- 统计功能单一，难以量化学习成果

### Proposed Solution

**MindReminder** 是一款桌面端的个人成长中枢，将科学的记忆算法与灵活的个性化配置相结合：

**核心功能：**

1. **智能可调复习算法**
   - 基于艾宾浩斯遗忘曲线的科学间隔重复
   - 全局复习频率系数（0.5x - 1.5x），适应个人节奏
   - 单个知识点独立系数调整
   - 基于自评分（1-5分）的动态间隔调整：低分（1-2分）知识点间隔缩短50%，高分知识点延长
   - 可自定义"记忆标准"（如30天后打4分以上视为掌握）
   - 达到记忆标准后的长期抽查机制

2. **日历式可视化界面**（最核心）
   - 清晰的日历视图，展示每天的新增知识点和复习任务
   - 每日摘要 + 点击查看详情
   - 过去记录和未来计划一目了然
   - 快速记录学习内容入口

3. **自评与进度追踪**
   - 简洁的1-5分自评系统
   - 每个知识点显示"下次复习时间"
   - 预测"多久可以记忆住"并可视化进度
   - 复习完成情况实时反馈

4. **三合一个人成长中枢**
   - **知识点管理**：以问题形式记录，支持复习和评分
   - **日记记录**：记录每天的思考和发生的事
   - **提醒事项**：管理未来待办和重要事件
   - 三者分类明确但统一管理，形成完整的成长档案
   - 可回顾某一天的完整状态：学了什么、做了什么、想了什么

5. **多维度统计分析**
   - 每日、每周、每月统计
   - 新增知识点数量、复习完成率、平均掌握度等指标
   - 学习趋势和习惯分析

**MVP技术特性（阶段1）：**

- 桌面应用（适配学习场景，主要在电脑前使用）
- 本地数据存储（数据隐私和掌控）
- 单机使用，无需联网

**使用流程：**

- 学习后立即记录知识点标题（问题形式）
- 晚上总结检查和补充记录
- 每天打开应用查看当日复习任务
- 对每个知识点自评打分
- 系统自动计算下次复习时间
- 通过日历和统计回顾学习历程

### Key Differentiators

**MindReminder 的独特竞争优势：**

1. **高度可定制的算法**
   - 首创"复习频率系数"概念，既有全局控制又支持单点精调
   - 用户真正掌控复习节奏，而非被工具控制
   - 可自定义记忆标准和长期抽查策略

2. **日历式可视化为核心**
   - 不同于列表式工具，日历视图让学习计划一目了然
   - 时间轴呈现让记忆管理变得直观和可掌控
   - 快速记录和摘要展示提升使用效率

3. **预测性记忆进度**
   - 不只是提醒复习，更告诉你"何时能真正记住"
   - 可自定义记忆标准，匹配个人学习目标
   - 掌握后的长期抽查保证持久记忆

4. **个人成长中枢定位**
   - 将知识、日记、提醒整合，形成完整的个人成长档案
   - 某一天的完整状态回顾：学了什么、做了什么、想了什么
   - 方便在一个地方管理所有事情

5. **桌面优先体验**
   - 针对"学习主要在电脑前"的场景深度优化
   - 丰富的UI和交互，不受移动端限制

6. **数据本地化（MVP阶段）**
   - 完全本地存储，无隐私顾虑
   - 用户对数据拥有100%掌控权

**为什么是现在：**
目标用户正在准备考试和面试，有迫切的学习效率需求，现有工具无法满足个性化和灵活性要求，市场存在明显的gap。

## Product Evolution Roadmap

### 阶段1：MVP - 桌面端本地存储（当前优先级）

**目标：** 验证核心价值，建立用户习惯

**核心功能：**

- 完整的智能复习算法（可调复习系数、自评打分、动态间隔）
- 日历式可视化界面
- 知识点管理（问题形式记录）
- 日记和提醒事项
- 多维度统计分析
- 本地数据存储

**技术架构考虑：**

- 采用模块化设计，为未来云端同步预留接口
- 数据模型设计考虑跨设备同步的需求
- 使用标准化的数据存储格式（如SQLite + JSON）

**成功指标：**

- 个人日常使用流畅
- 复习提醒准确有效
- 学习效率和记忆留存率提升

### 阶段2：跨设备同步

**目标：** 实现多设备数据同步，提升便利性

**新增功能：**

- 云服务器数据同步
- 多台桌面设备数据一致性
- 冲突解决机制
- 离线编辑 + 自动同步

**技术实现：**

- 云服务器架构（如使用阿里云、腾讯云等）
- RESTful API 或 WebSocket 实时同步
- 增量同步减少带宽消耗
- 端到端加密保证数据安全

**用户价值：**

- 在家里和办公室的电脑间无缝切换
- 数据备份，防止丢失
- 保持本地存储选项，给用户选择权

### 阶段3：移动端应用

**目标：** 扩展使用场景，完善产品生态

**核心功能：**

- 保留桌面端大部分功能
- **重点强化：** 通知功能（复习提醒推送）
- 快速记录知识点
- 查看今日复习任务
- 完整复习和评分功能
- 查看统计和日历
- 日记和提醒管理

**移动端独特价值：**

- 随时随地接收复习通知
- 碎片时间快速复习
- 临时学习内容即时记录
- 通勤路上完成复习任务

**跨平台协同：**

- 桌面端：深度学习和内容管理的主战场
- 移动端：通知提醒和碎片化复习的补充
- 数据实时同步，体验无缝衔接

### 长期愿景

**成为个人知识管理和成长的一站式平台：**

- 智能算法持续优化（AI辅助推荐复习内容）
- 知识图谱可视化（知识点关联）
- 社区分享功能（可选，隐私优先）
- 与其他学习工具集成（如笔记软件、思维导图）
- 跨平台生态完善（Windows、macOS、iOS、Android）

## Target Users

### Primary User Segment 1: 在校大学生

**角色画像 - 李明（大三计算机专业学生）**

**背景与日常：**

- 22岁，计算机专业大三学生，正在准备考研和找实习
- 每天主要在电脑前学习：上网课、写代码、看专业书、背英语单词
- 学习内容广泛：编程（数据结构、算法）、专业课（操作系统、数据库）、考研数学、英语等
- 学习强度不稳定：考试周可能一天记录几十到上百个知识点，平时可能十几个

**当前问题体验：**

- **核心痛点**：学了就忘，没有系统的复习机制
- **考前焦虑**：临近考试时完全不记得这学期学了什么，不知道该复习哪些内容
- **效率低下**：反复重新学习同样的知识点，浪费时间
- **缺乏掌控感**：不清楚自己的学习进度和记忆状态

**使用场景：**

- **学习后立即记录**：上完一节课、看完一个教程后，快速记录今天学到的知识点
- **晚上复习时段**：每晚固定时间打开 MindReminder，查看今天的复习任务
- **考前冲刺**：通过日历查看这学期学习的所有内容，系统性复习

**成功标准：**

- **短期**：考试取得高分，答题时能想起学过的知识点
- **长期**：真正掌握技能，知识能在实际项目中应用

**对产品的期望：**

- 看到复习提醒时感觉是"友好的督促"而非压力
- 清晰知道今天要复习什么、已经学过什么
- 看到学习成果的可视化，获得成就感和动力

**当前问题体验：**

- **最大担心**：
  - 准备不充分就去面试
  - 面试时突然忘记关键知识点，卡壳尴尬
  - 投入大量时间学习但效果不明显

**使用场景：**

**成功标准：**

- **最重要**：面试时真的用上了记住的知识点，顺利通过
- **持续价值**：技能真正提升，不只是应付面试
- **心理准备**：面试前看到自己学过的所有知识点，有信心

**对产品的期望：**

- 清晰的复习计划，不遗漏任何重要知识点
- 统计数据证明自己的进步，增强信心
- 灵活的复习频率调整，适应工作忙碌程度

### User Journey（用户旅程）

**阶段1：发现与初次使用**

- **触发点**：考试临近/面试在即，意识到需要科学的复习工具
- **下载安装**：快速安装桌面应用
- **首次体验**：
  - 打开应用，看到简洁的日历界面
  - 有默认的复习算法设置（可调整但不必强制配置）
  - 可选查看简单教程（"快速开始指南"），也可以直接跳过
  - 立即记录第一个知识点，体验记录流程

**阶段2：建立习惯（第1周）**

- **学习后记录**：
  - 每次学习后立即打开 MindReminder 快速记录
  - 用问题形式记录知识点标题
  - 晚上总结时补充和检查
- **首次复习体验**：
  - 第2天看到第一个复习提醒
  - 点开查看需要复习的知识点列表
  - 逐个回忆并自评打分（1-5分）
  - 看到系统自动计算的"下次复习时间"

- **第一个"啊哈时刻"**：
  - 看到日历上开始出现连续的学习记录
  - 发现经过复习真的记住了之前容易忘的内容
  - 或者看到第一周统计：记录了X个知识点，完成了Y次复习

**阶段3：深度使用（2-4周）**

- **日常流程固化**：
  - 学习 → 立即记录 → 晚上复习任务 → 打分 → 查看统计
  - 成为日常习惯的一部分
- **个性化调整**：
  - 发现某些知识点提醒频率太高，调整复习系数
  - 尝试使用日记功能记录学习心得
  - 使用提醒事项功能管理其他待办

- **持续"啊哈时刻"**：
  - 某个知识点达到"记忆标准"，看到进度条完成
  - 月度统计显示记录了数百个知识点，复习完成率90%+
  - 翻看日历回顾整个月的学习历程，很有成就感

**阶段4：成功验证（考试/面试时）**

- **考前/面前准备**：
  - 通过日历快速浏览所有学过的知识点
  - 针对性复习标记为"不太熟悉"的内容
  - 看到完整的学习记录，心里有底，信心增强

- **实际应用**：
  - 考试/面试中遇到题目，能想起在 MindReminder 中复习过
  - 顺利答题/回答面试问题
  - **最大的成功时刻**：真的用上了记住的知识点！

**阶段5：长期价值（3个月+）**

- **习惯养成**：
  - MindReminder 成为学习工作流的标配工具
  - 不仅用于考试面试，也用于日常技能积累
- **个人成长档案**：
  - 回顾几个月的学习历程
  - 看到自己的知识体系逐渐完善
  - 日记功能记录了成长的点滴

- **持续优化**：
  - 根据使用习惯调整复习策略
  - 期待云同步和移动端功能（阶段2、3）

## Success Metrics

### MVP阶段（个人使用 - 前3个月）

**用户成功指标：**

1. **使用习惯养成**
   - **持续使用天数**：连续使用天数作为习惯养成的标志
   - **每日完成复习任务**：按时完成每天的复习计划
   - 目标：建立稳定的学习和复习习惯

2. **知识留存率提升**
   - **记住更多知识点**：通过复习真正记住之前容易忘的内容
   - **自评分数改善**：同一知识点在多次复习中评分逐渐提高
   - **考试/面试验证**：在实际场景中能想起复习过的知识点

3. **复习完成情况**
   - **复习完成率**：按计划完成复习任务的比例
   - **复习及时性**：在提醒时间点附近完成复习

4. **用户"啊哈时刻"（关键体验指标）**
   - **看到日历上完整的学习记录**：可视化学习历程
   - **发现真的记住了**：复习时能够回忆起知识点
   - **看到统计数据发现进步**：通过数据看到自己的成长

**产品体验指标：**

1. **日历可视化价值**
   - 能清晰看到每天的学习记录和复习计划
   - 快速定位某一天或某段时间的学习内容

2. **复习提醒有效性**
   - 提醒时间准确
   - 提醒内容清晰（知道今天要复习什么）
   - 提醒感觉是"友好督促"而非压力

3. **灵活性满足**
   - 能够根据个人情况调整复习频率系数
   - 单个知识点可以独立调整
   - 算法参数可自定义（记忆标准、间隔调整等）

**最终成功验证：**

- **知识真正掌握**：不只是短期记住，而是真正理解和掌握技能

## MVP Scope

### Core Features（核心功能）

**1. 智能可调复习算法**

- ✅ **MVP 必需**
- 基于艾宾浩斯遗忘曲线的间隔重复算法
- 全局复习频率系数（0.5x - 1.5x）调整
- 单个知识点独立系数调整
- **人性化自评系统**（替代1-5分）：
  - 😟 忘记了
  - 🤔 记得一点
  - 😐 记得一般
  - 😊 记得还可以
  - 🎯 非常熟悉
- 基于自评的动态间隔调整（"忘记了"/"记得一点"的知识点间隔缩短50%，"非常熟悉"的延长）
- 可自定义记忆标准（如30天后达到"记得还可以"以上视为掌握）
- 达到记忆标准后的长期抽查机制
- **价值**：这是解决"学了就忘"核心问题的科学基础，人性化标签让评分更直观友好

**2. 日历式可视化界面**

- ✅ **MVP 必需（最核心）**
- 清晰的日历视图，展示每天的学习记录
- 显示每个知识点的"下次复习时间"
- 每日摘要展示：
  - 今天新增的知识点数量和列表
  - 今天要复习的知识点数量和列表
- 点击日期查看详情功能
- 快速记录知识点入口（快捷键/按钮）
- **MVP重点**：优先保证"看到学习记录"和"未来复习时间"可视化清晰直观
- **价值**：让学习历程和复习计划一目了然，创造第一个"啊哈时刻"

**3. 自评与基础进度追踪**

- ✅ **MVP 必需**
- 人性化自评标签系统（见功能1）
- 每个知识点显示"下次复习时间"
- 复习完成情况反馈（今日完成X个，剩余Y个）
- **MVP简化**：精美的"多久能记住"可视化进度条可以简化，先显示基本信息即可

**4. 三合一个人成长中枢**

- ✅ **MVP 必需**
- **知识点管理**（最核心）：
  - 以问题形式记录知识点标题
  - 支持复习和人性化评分
  - 知识点分类/标签功能
  - 知识点搜索（基础）
  - 显示每个知识点的复习历史和评分趋势
- **日记记录**：
  - 记录每天的思考、总结和发生的事
  - 在日历上与知识点分类展示（不同视觉样式）
  - 支持富文本或 Markdown 格式
- **提醒事项**：
  - 管理未来待办和重要事件
  - 在日历上显示提醒（不同于知识点的视觉样式）
  - 提醒时间到达时通知
- **价值**：形成完整的每日成长档案，可回顾某一天的完整状态（学了什么、做了什么、想了什么）

**5. 本地数据存储**

- ✅ **MVP 必需**
- 使用 SQLite 或类似轻量级数据库
- 数据模型设计考虑未来云同步扩展
- 数据导出功能（JSON/CSV格式，用于备份）
- 数据导入功能（从备份恢复）

### Out of Scope for MVP（MVP 不包含的功能）

**阶段性延后的功能：**

1. **完整的统计分析功能**
   - **状态**：可在 MVP 最后阶段添加，但不是初期必需
   - 每日/每周/每月详细统计图表
   - 新增知识点数量趋势
   - 复习完成率趋势
   - 平均掌握度分析
   - 学习习惯和模式分析
   - **MVP策略**：MVP 可以先提供基础的"今日统计"，详细统计功能在后期完善
   - **理由**：统计是辅助功能，核心价值在于复习机制本身

2. **高级进度预测可视化**
   - 精美的"多久可以记忆住"进度条动画
   - 知识点掌握进度的可视化图表
   - 记忆曲线可视化
   - **MVP策略**：显示基本文字信息即可（如"预计X天后掌握"），视觉呈现可以"工作但不完美"
   - **理由**：功能存在更重要，视觉优化可以迭代

3. **高级知识点管理功能**
   - 知识点之间的关联和依赖关系
   - 知识图谱可视化
   - 批量操作（批量编辑、删除、调整）
   - 高级搜索和筛选（多条件组合）
   - **理由**：MVP 阶段基础功能够用，高级功能按需求优先级添加

4. **跨设备同步（阶段2）**
   - 云服务器数据同步
   - 多设备数据一致性
   - 冲突解决机制
   - **理由**：MVP 先验证核心价值，同步功能在阶段2添加

5. **移动端应用（阶段3）**
   - iOS/Android 应用
   - 移动端推送通知
   - 碎片化复习体验
   - **理由**：先完善桌面端体验，再扩展移动端

6. **高级功能（未来版本）**
   - AI 辅助推荐复习内容
   - 智能间隔调整（基于用户历史数据自动优化）
   - 社区分享功能
   - 与其他学习工具集成（笔记软件、思维导图等）
   - 多语言支持
   - **理由**：这些是增值功能，不影响核心价值实现

### MVP Success Criteria（MVP 成功标准）

**用户验证指标：**

1. **能够每天持续使用**
   - 连续使用7天以上
   - 每天至少记录1个知识点或完成1次复习

2. **复习机制有效**
   - 使用2周后，发现真的记住了之前容易忘的内容
   - 自评从"忘记了"/"记得一点"提升到"记得还可以"/"非常熟悉"

3. **日历价值明确**
   - 通过日历能清晰看到学习历程和复习计划
   - 能快速定位某一天或某段时间的学习内容

4. **三合一体验流畅**
   - 知识点、日记、提醒三个功能都能正常使用且有价值
   - 能够回顾某一天的完整状态

5. **灵活性满足需求**
   - 能够根据个人情况调整复习频率系数
   - 调整后复习负担合理（不会太多或太少）

**技术验证指标：**

1. **算法准确性**
   - 复习提醒时间计算正确
   - 基于人性化评分的间隔调整符合预期

2. **性能流畅**
   - 记录100+个知识点时，界面依然流畅
   - 日历切换和数据加载快速响应

3. **数据可靠**
   - 本地存储稳定，不丢失数据
   - 数据导出导入功能正常

**体验验证指标：**

1. **首次上手容易**
   - 无需教程也能在5分钟内记录第一个知识点
   - 界面直观，人性化评分标签易于理解

2. **"啊哈时刻"触发**
   - 使用一周后能看到完整的学习记录
   - 首次复习时发现真的记住了内容

3. **提醒友好不压力**
   - 看到复习提醒时感觉是督促而非负担
   - 复习计划清晰但不焦虑

**MVP 通过标准：**

- ✅ 如果以上指标都达到，且愿意每天使用，MVP 成功
- ✅ 如果核心问题（学了就忘）得到明显改善，继续投入开发
- ✅ 如果考试/面试时真的用上了记住的知识点，最终成功验证
