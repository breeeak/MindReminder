# Product Requirements Document - MindReminder

**Author:** Administrator
**Date:** 2025-12-13

## 执行摘要

MindReminder 是一款面向自主学习者的桌面应用，旨在通过科学的间隔重复方法帮助用户巩固知识、养成高效复习习惯。产品将知识管理、日记记录和提醒事项整合为统一的个人成长中枢，采用本地数据存储，为用户提供完全的数据掌控权和隐私保护。

**核心问题：** 知识学习者面临"学了就忘"的普遍痛点。尽管科学研究证明间隔重复能有效对抗遗忘曲线，但市面上缺少真正适合个人学习节奏的桌面复习工具。现有工具复习频率固定，无法根据个人情况调整，导致复习量过大或不足。同时，知识点、日常记录、待办事项分散在不同工具中，无法形成完整的学习画像。

**目标用户：** 主要服务两大用户群体：
1. **在校大学生** - 正在准备考研和找实习，每天在电脑前学习，需要记忆大量专业知识点
2. **职场技能提升者** - 准备跳槽面试，需要系统地记忆技术知识和新技术栈

这些用户主要在电脑前学习和工作，需要一个既科学又灵活的复习管理系统。

### 产品独特价值

MindReminder 通过以下核心差异化解决现有工具的痛点：

1. **高度可定制的复习算法**
   - 首创"复习频率系数"概念（0.5x-1.5x），支持全局控制和单点精调
   - 用户真正掌控复习节奏，而非被工具控制
   - 基于自评的动态间隔调整，匹配个人学习能力

2. **日历式可视化核心界面**
   - 不同于列表式工具，日历视图让学习计划和复习任务一目了然
   - 时间轴呈现让记忆管理变得直观和可掌控
   - 过去记录和未来计划清晰展现，创造"啊哈时刻"

3. **人性化自评系统**
   - 使用表情符号（😟 忘记了 → 🎯 非常熟悉）替代传统1-5分数字评分
   - 更直观友好，降低认知负担
   - 让评分过程更自然、更符合人的思维方式

4. **预测性记忆进度**
   - 不只是提醒复习，更告诉用户"何时能真正记住"
   - 可自定义记忆标准（如30天后达到"记得还可以"视为掌握）
   - 掌握后的长期抽查机制保证持久记忆

5. **个人成长中枢定位**
   - 将知识点、日记、提醒事项整合在一个应用中
   - 可回顾某一天的完整状态：学了什么、做了什么、想了什么
   - 形成完整的个人成长档案

6. **桌面优先体验**
   - 针对"学习主要在电脑前"的场景深度优化
   - 丰富的UI和交互，不受移动端限制
   - 本地数据存储，100%数据掌控权

## 项目分类

**技术类型：** Desktop App（桌面应用）
**领域：** Edtech（教育技术）
**复杂度：** Medium（中等）
**项目背景：** Greenfield - 全新项目

**技术特征：**
- 跨平台桌面应用（Windows、macOS）
- 本地数据存储（SQLite + JSON）
- 单机使用，无需联网（MVP阶段）
- 模块化设计，为未来云端同步预留接口

**领域考虑：**
作为教育技术产品，需要关注：
- 学生隐私保护（COPPA/FERPA合规性考虑）
- 可访问性要求（WCAG标准）
- 内容指南和学习标准
- 用户数据安全和隐私

**演进路线：**
产品将分阶段演进：
- **阶段1（MVP）**：桌面端本地存储
- **阶段2**：跨设备云端同步
- **阶段3**：移动端应用（iOS/Android）

## 成功标准

### 用户成功

用户成功的核心在于解决"学了就忘"的痛点，让学习者真正记住并掌握知识。

**核心成功时刻：**

1. **第一周 - 习惯建立**
   - 用户能连续使用7天以上
   - 每天至少记录1个知识点或完成1次复习
   - 看到日历上开始出现连续的学习记录

2. **第二周 - 价值验证**
   - 首次复习时发现真的记住了之前容易忘的内容
   - 自评从"忘记了😟"/"记得一点🤔"提升到"记得还可以😊"/"非常熟悉🎯"
   - 复习完成率达到合理水平（不会感到负担过重）

3. **第一个月 - 成就感建立**
   - 通过日历能看到完整的学习轨迹
   - 统计数据显示记录了多个知识点，完成了多次复习
   - 产生"原来我学了这么多"的成就感

4. **最终验证 - 实际应用**
   - 在考试或面试中真的用上了记住的知识点
   - 关键时刻能回忆起重要知识
   - 知识不只是短期记住，而是真正掌握

**产品体验成功：**
- **日历可视化价值**：能清晰看到学习历程和复习计划
- **复习提醒有效性**：提醒时间准确，内容清晰，感觉是"友好督促"而非压力
- **灵活性满足**：能够根据个人情况调整复习频率，复习负担合理
- **首次上手容易**：无需教程也能在5分钟内记录第一个知识点

### 业务成功

**MVP阶段（前3个月）- 核心价值验证：**

1. **持续使用验证**
   - 个人能够每天持续使用
   - 连续使用天数持续增长
   - 成为日常学习工作流的一部分

2. **功能价值验证**
   - 复习机制真的有效，知识留存率明显提升
   - 日历可视化确实带来"啊哈时刻"
   - 三合一功能（知识+日记+提醒）被实际使用

3. **最终目标验证**
   - 在实际考试或面试中真的用上了记住的知识点
   - 学习效率和记忆留存率提升
   - 愿意继续长期使用

**长期目标（12个月+ - 如果MVP成功）：**

1. **用户增长**
   - 目标用户群体（大学生、职场人士）的用户数量增长
   - 用户获取渠道有效性

2. **用户留存**
   - 次日留存率、7日留存率、30日留存率
   - 活跃用户比例（DAU/MAU）
   - 用户平均使用时长

3. **用户参与**
   - 平均每周使用频率
   - 平均每日记录知识点数量
   - 复习完成率（整体用户平均）

4. **用户满意度**
   - 用户推荐意愿（NPS）
   - 关键功能满意度评分
   - 用户反馈和功能请求

### 技术成功

**核心技术要求（MVP）：**

1. **算法准确性**
   - 复习提醒时间计算正确，符合艾宾浩斯遗忘曲线
   - 基于人性化评分（😟→🎯）的间隔调整符合预期
   - 记忆标准判断准确（如30天后达到😊以上视为掌握）
   - 长期抽查机制正常触发

2. **性能流畅**
   - 记录100+个知识点时，界面依然流畅响应
   - 日历切换和数据加载快速（<1秒）
   - 应用启动时间合理（<3秒）
   - 搜索和筛选功能响应快速

3. **数据可靠性**
   - 本地存储稳定，不丢失数据
   - 数据导出导入功能正常（JSON/CSV格式）
   - 应用崩溃后数据可恢复
   - 数据完整性校验

4. **可扩展性**
   - 数据模型设计考虑未来云同步扩展
   - 模块化架构便于功能迭代
   - 使用标准化的数据存储格式（SQLite + JSON）
   - 为跨设备同步预留接口

5. **跨平台兼容性**
   - Windows和macOS平台稳定运行
   - 界面适配不同屏幕分辨率
   - 系统通知正常工作

## 产品范围

### MVP - 最小可行产品

**必须工作才有用的核心功能：**

#### 1. 智能可调复习算法（核心功能）
- 基于艾宾浩斯遗忘曲线的间隔重复算法
- **全局复习频率系数**（0.5x - 1.5x）调整
- **单个知识点独立系数**调整
- **人性化自评系统**：
  - 😟 忘记了
  - 🤔 记得一点
  - 😐 记得一般
  - 😊 记得还可以
  - 🎯 非常熟悉
- 基于自评的动态间隔调整（😟/🤔的知识点间隔缩短50%，🎯的延长）
- 可自定义**记忆标准**（如30天后达到😊以上视为掌握）
- 达到记忆标准后的**长期抽查机制**

#### 2. 日历式可视化界面（最核心功能）
- 清晰的日历视图，展示每天的学习记录
- 显示每个知识点的"下次复习时间"
- **每日摘要展示**：
  - 今天新增的知识点数量和列表
  - 今天要复习的知识点数量和列表
- 点击日期查看详情功能
- **快速记录知识点入口**（快捷键/按钮）

#### 3. 知识点管理
- 以**问题形式**记录知识点标题
- 支持复习和人性化评分
- **知识点分类/标签**功能
- **知识点搜索**（基础）
- 显示每个知识点的**复习历史和评分趋势**
- 复习完成情况反馈（今日完成X个，剩余Y个）

#### 4. 日记和提醒功能
- **日记记录**：
  - 记录每天的思考、总结和发生的事
  - 在日历上与知识点分类展示（不同视觉样式）
  - 支持富文本或Markdown格式
- **提醒事项**：
  - 管理未来待办和重要事件
  - 在日历上显示提醒（不同于知识点的视觉样式）
  - 提醒时间到达时通知

#### 5. 本地数据存储
- 使用SQLite或类似轻量级数据库
- 数据模型设计考虑未来云同步扩展
- **数据导出功能**（JSON/CSV格式，用于备份）
- **数据导入功能**（从备份恢复）

**MVP简化策略：**
- 基础统计即可（今日统计），详细统计图表后期完善
- "多久能记住"先显示基本文字信息，精美进度可视化后期优化
- 高级知识点管理功能（关联、批量操作、高级搜索）后期添加

### 暂不包含（MVP之后）

#### 阶段性延后的功能：

1. **完整的统计分析功能**
   - 每日/每周/每月详细统计图表
   - 新增知识点数量趋势
   - 复习完成率趋势
   - 平均掌握度分析
   - 学习习惯和模式分析
   - **MVP策略**：提供基础"今日统计"，详细统计后期完善

2. **高级进度预测可视化**
   - 精美的"多久可以记忆住"进度条动画
   - 知识点掌握进度的可视化图表
   - 记忆曲线可视化
   - **MVP策略**：显示基本文字信息（如"预计X天后掌握"），视觉呈现后期优化

3. **高级知识点管理功能**
   - 知识点之间的关联和依赖关系
   - 知识图谱可视化
   - 批量操作（批量编辑、删除、调整）
   - 高级搜索和筛选（多条件组合）

4. **跨设备同步（阶段2）**
   - 云服务器数据同步
   - 多设备数据一致性
   - 冲突解决机制

5. **移动端应用（阶段3）**
   - iOS/Android应用
   - 移动端推送通知
   - 碎片化复习体验

6. **高级功能（未来版本）**
   - AI辅助推荐复习内容
   - 智能间隔调整（基于用户历史数据自动优化）
   - 社区分享功能
   - 与其他学习工具集成
   - 多语言支持

### 演进路线（阶段2-3）

**阶段2：跨设备云端同步（6-12个月）**
- 云服务器数据同步
- 多台桌面设备数据一致性
- 增量同步减少带宽消耗
- 端到端加密保证数据安全
- 保持本地存储选项，给用户选择权

**阶段3：移动端应用（12个月+）**
- iOS和Android应用
- 重点强化通知功能（复习提醒推送）
- 快速记录和碎片化复习
- 与桌面端数据实时同步
- 跨平台无缝体验

**长期愿景：**
- 智能算法持续优化（AI辅助）
- 知识图谱可视化
- 成为个人知识管理和成长的一站式平台

## 用户旅程

### 旅程 1：李明 - 从考前焦虑到胸有成竹

**李明的困境**

李明是一名大三计算机专业学生，距离考研只剩下6个月。每天晚上，他坐在电脑前，打开各种笔记软件、Anki卡片、在线课程平台，感觉自己一直在"学习"，但内心却越来越焦虑。上周末，他想复习一下数据结构的红黑树，突然发现一个月前学的AVL树已经完全记不起来了。更可怕的是，他甚至记不清这学期到底学了哪些知识点。

一天晚上，李明在整理学习笔记时偶然发现了 MindReminder。"又一个记忆工具？"他心想，但看到日历式界面和可调节的复习频率，他决定试一试。

**转折点**

第一周，李明开始使用 MindReminder。每天上完课或看完教程后，他会立即打开应用，用几分钟快速记录今天学到的关键知识点："什么是红黑树的平衡条件？"、"HashMap的扩容机制？"。晚上10点，他打开应用查看今天的复习任务，只有3个知识点需要复习。他逐个回忆，用表情符号打分：第一个😐记得一般，第二个🤔记得一点，第三个😊记得还可以。整个过程只花了15分钟，但他感觉很充实。

第二周的某个晚上，李明完成复习后，习惯性地点开日历查看。他惊讶地发现，日历上已经连续记录了12天的学习痕迹，每天都有新增的知识点和完成的复习任务。"原来我已经记录了50多个知识点了！"这是他第一次清晰地看到自己的学习轨迹。

**关键时刻**

一个月后，李明参加了一次模拟面试。面试官问："请解释一下MySQL的索引原理。"李明的脑海中立刻浮现出他在 MindReminder 中反复复习的那个知识点，包括他两周前标记为😊的B+树结构。他流畅地回答了问题，面试官点头表示满意。

考研前一周，李明通过 MindReminder 的日历功能快速浏览了整个学期的学习内容。300多个知识点，大部分已经达到🎯非常熟悉的程度。他第一次对即将到来的考试感到踏实和自信。

**新的现实**

考研那天，李明在考场上遇到的大部分题目都能迅速联想到他在 MindReminder 中复习过的知识点。写完最后一道算法题，他知道自己已经成功了。三个月后,李明收到了心仪学校的录取通知。他打开 MindReminder，在日记功能中写下："感谢你陪我度过这段时光。我不只是考上了研究生，更重要的是，我学会了如何真正掌握知识。"

如今，李明仍然每天使用 MindReminder，不再是为了考试，而是为了持续学习和成长。

### 旅程 2：王芳 - 从面试焦虑到Offer收割机

**王芳的挑战**

王芳在一家中等规模的互联网公司工作了3年，技术能力不错，但一直想跳槽到更大的平台。她给自己定下目标：半年内拿到大厂Offer。问题是，她需要记忆大量的"八股文"——前端框架原理、浏览器机制、算法题、设计模式……内容既枯燥又容易忘。

她试过各种方法：在印象笔记里整理面试题、在Anki里做卡片、甚至买了付费课程。但每天下班后只有2-3小时学习时间，她总觉得时间不够用，学过的东西很快就忘，心里越来越没底。一次周末模拟面试，面试官问到"React的Fiber架构原理"，明明上周刚看过，她却怎么也想不起来。那一刻，她感到深深的挫败感。

**遇见希望**

某个周末，王芳在技术社区看到有人推荐 MindReminder。"又一个背单词工具的变种？"她有些怀疑，但看到"可自定义复习频率"和"日历式管理"，她决定试试。

第一天，王芳花了半小时整理当天学习的内容，在 MindReminder 中记录了10个问题："React的虚拟DOM是如何工作的？"、"浏览器的事件循环机制？"。系统提示她明天需要复习。她点开设置，将全局复习频率调整到0.8x（因为工作日时间紧张），然后把几个特别难记的知识点单独调到1.2x（需要更频繁复习）。这种掌控感让她安心了不少。

**积累与突破**

## 桌面应用技术要求

### 技术架构概览

MindReminder 采用 Electron 框架构建跨平台桌面应用，使用现代 Web 技术栈实现丰富的用户界面，同时保持原生桌面应用的性能和用户体验。

**核心技术栈：**
- **应用框架**：Electron（跨平台支持 Windows、macOS）
- **前端框架**：React/Vue（根据开发偏好选择）
- **本地数据库**：SQLite（结构化数据存储）
- **配置存储**：JSON 文件（用户配置和设置）
- **UI组件**：现代化组件库（确保日历视图等核心界面美观流畅）

### 平台支持要求

**目标平台（MVP）：**
- **Windows**：Windows 10 及以上版本
- **macOS**：macOS 10.14 (Mojave) 及以上版本

**平台特定适配：**
- 遵循各平台的 UI 设计规范
- 窗口行为符合平台惯例（如 macOS 的红绿灯按钮）
- 文件路径处理跨平台兼容
- 平台特定快捷键映射（Ctrl vs Cmd）

### 系统集成功能

**1. 系统托盘集成**
- 应用最小化到系统托盘
- 托盘图标显示未读复习任务数量（可选）
- 托盘右键菜单：
  - 打开主窗口
  - 快速记录知识点
  - 查看今日复习任务
  - 退出应用

**2. 开机自启动**
- 用户可在设置中开启/关闭开机自启动
- 启动后最小化到托盘（不干扰用户）
- 默认关闭，由用户主动开启

**3. 全局快捷键**
- **Ctrl/Cmd + Shift + M**：快速打开应用主窗口
- **Ctrl/Cmd + Shift + N**：全局快速记录知识点（弹出小窗口）
- 快捷键可在设置中自定义
- 避免与常用应用快捷键冲突

**4. 系统通知**
- 复习提醒通知（可在设置中配置时间）
- 通知内容：今日待复习知识点数量
- 点击通知直接打开应用并显示复习列表
- 通知频率可配置（避免打扰）

### 离线优先架构

**完全离线运行（MVP）：**
- 无需联网即可使用所有功能
- 所有数据存储在本地
- 不依赖任何云服务或外部API

**数据存储设计：**
- **SQLite 数据库**：
  - 知识点数据（ID、标题、内容、标签、评分历史）
  - 复习记录（复习时间、评分、下次复习时间）
  - 日记记录
  - 提醒事项
- **JSON 配置文件**：
  - 用户设置（全局复习频率系数、记忆标准等）
  - 应用配置（窗口大小、位置、主题等）
- **数据文件位置**：
  - Windows：`%APPDATA%/MindReminder/`
  - macOS：`~/Library/Application Support/MindReminder/`

**为未来同步预留设计：**
- 数据模型包含 `id`（UUID）、`created_at`、`updated_at`、`sync_status` 字段
- 模块化数据访问层，便于未来替换为云同步逻辑
- 数据导出/导入功能验证数据序列化方案

### 性能优化要求

**启动性能：**
- 应用冷启动时间 < 3 秒
- 热启动（从托盘恢复）< 1 秒
- 使用 Electron 启动优化技术（如延迟加载非关键模块）

**运行性能：**
- 记录 100+ 个知识点时，界面依然流畅（60 FPS）
- 日历切换和数据加载 < 1 秒
- 搜索响应时间 < 500 毫秒
- 内存占用合理（< 300MB 在正常使用下）

**UI 渲染优化：**
- 日历视图使用虚拟滚动技术
- 知识点列表使用虚拟列表（处理大量数据）
- 图表和统计按需加载
- 图片和富文本内容懒加载

### 数据可靠性与备份

**自动备份机制：**
- 每日自动备份数据库到本地备份文件夹
- 保留最近 7 天的备份文件
- 备份文件命名：`backup_YYYY-MM-DD.sqlite`
- 用户可手动触发备份

**崩溃恢复：**
- 应用异常退出后，下次启动时恢复未保存数据
- SQLite 事务处理保证数据一致性
- 定期保存用户操作状态

**数据完整性：**
- SQLite 数据库完整性检查
- 启动时验证数据文件
- 数据导入时验证格式和完整性

**数据导出/导入：**
- 导出格式：JSON、CSV
- 导入支持：从导出文件恢复数据
- 支持选择性导出（如只导出某个分类的知识点）

### 窗口管理

**主窗口：**

- 始终置顶
- 快速输入知识点标题

### 风险缓解策略

**技术风险识别与应对：**

**风险1：复习算法的准确性和有效性**
- **风险等级：** 高（直接影响核心价值）
- **风险描述：** 算法实现不当可能导致复习频率不合理，用户负担过重或复习不足
- **缓解策略：**
  - 基于成熟的艾宾浩斯遗忘曲线理论
  - 参考Anki、SuperMemo等成熟工具的算法实践
  - 实现灵活的参数调整机制（全局和单点系数）
  - 提供合理的默认配置（经过测试验证）
- **验证方法：**
  - 个人长期使用验证（至少2-3个月）
  - 收集自评数据和复习完成率
  - 根据反馈持续调优参数
- **应急方案：** 如果算法效果不佳，可快速调整参数或简化算法逻辑

**风险2：日历界面的性能（大量数据时）**
- **风险等级：** 中（影响用户体验）
- **风险描述：** 当用户记录数百个知识点时，日历渲染可能变慢
- **缓解策略：**
  - 使用虚拟滚动技术（只渲染可见区域）
  - 按需加载数据（延迟加载非当前月份数据）
  - 数据分页和缓存策略
  - 优化日历组件性能
- **验证方法：**
  - 模拟100+、500+、1000+知识点场景测试
  - 性能监控和优化
- **性能目标：** 即使1000个知识点，日历切换仍<1秒

**风险3：Electron应用的体积和启动速度**
- **风险等级：** 中（影响用户体验）
- **风险描述：** Electron应用通常体积较大，启动较慢
- **缓解策略：**
  - 采用代码分割和Tree Shaking
  - 延迟加载非关键模块
  - 使用Electron优化最佳实践
  - 打包时压缩和优化资源
- **性能目标：**
  - 冷启动时间 < 3秒
  - 安装包体积 < 150MB
  - 运行时内存占用 < 300MB

**风险4：跨平台兼容性问题**
- **风险等级：** 中
- **风险描述：** Windows和macOS平台差异可能导致功能或UI问题
- **缓解策略：**
  - 使用跨平台兼容的库和组件
  - 平台特定代码隔离和条件编译
  - 在两个平台上持续测试
- **验证方法：** 每个版本都在Windows和macOS上测试

**市场风险识别与应对：**

**风险1：目标用户是否真正需要这样的工具**
- **风险等级：** 高（影响产品存在价值）
- **风险描述：** 目标用户可能已经有自己的学习方法，不愿改变
- **缓解策略：**
  - MVP阶段重点个人使用验证（先验证对自己有用）
  - 设计简单的上手流程（5分钟内能记录第一个知识点）
  - 提供合理的默认配置（无需复杂设置即可使用）
  - 强调核心价值：解决"学了就忘"的痛点
- **验证指标：**
  - 连续使用7天以上
  - 复习完成率 > 70%
  - 考试/面试时真的用上了
- **应急方案：** 如果验证失败，重新评估产品定位或调整功能

**风险2：与现有工具（如Anki）竞争**
- **风险等级：** 中
- **风险描述：** Anki等工具已经有成熟的用户群体和生态
- **差异化策略：**
  - **日历可视化**：Anki没有的核心差异化功能
  - **灵活复习系数**：更个性化的复习节奏控制
  - **三合一功能**：知识点+日记+提醒整合
  - **桌面优先体验**：针对电脑学习场景深度优化
  - **人性化评分**：表情符号比数字更直观友好
- **产品定位：**
  - 不是Anki的直接替代品
  - 面向"主要在电脑前学习"的特定场景
  - 为想要更灵活、更直观工具的用户提供新选择

**风险3：用户获取和增长困难**
- **风险等级：** 中（阶段2-3时的挑战）
- **风险描述：** MVP验证成功后，如何让更多用户知道和使用
- **缓解策略（未来考虑）：**
  - 技术社区推广（GitHub、Reddit、知乎、V2EX等）
  - 内容营销（撰写学习方法和效率提升的文章）
  - 口碑传播（产品真正有价值，用户会自然推荐）
  - 教育机构合作（未来可能性）
- **MVP阶段：** 不考虑增长，专注产品价值验证

**资源风险识别与应对：**

**风险1：开发资源不足或进度延迟**
- **风险等级：** 高
- **风险描述：** 开发人员时间有限，可能无法按计划完成所有功能
- **降级方案（按优先级执行）：**

**降级方案1 - 简化非核心功能：**
- 保留：知识点管理 + 复习算法 + 日历视图
- 简化：日记和提醒功能延后到1.1版本
- 简化：统计功能只保留基础数据（今日统计）
- 预计节省开发时间：20-30%

**降级方案2 - 进一步简化日历功能：**
- 保留：基础日历视图和当日任务列表
- 简化：日历详细交互和可视化效果
- 延后：高级日历功能（搜索、筛选等）
- 预计节省开发时间：10-15%

**降级方案3 - 最小可行MVP：**
- **绝对核心保留：**
  - 知识点记录和管理（问题形式）
  - 复习算法（基于艾宾浩斯）
  - 人性化自评系统（😟→🎯）
  - 复习提醒和任务列表
  - 简单日历视图（列表形式）
  - 本地数据存储（SQLite）
- **全部延后到1.1：**
  - 日历详细可视化
  - 复习频率系数调整
  - 日记和提醒功能
  - 统计功能
  - 数据导出/导入
- **价值验证：** 即使是最小MVP，仍然能验证核心算法和复习机制的有效性
- **预计开发时间：** 减少至1.5-2个月

**风险2：UI/UX设计资源不足**
- **风险等级：** 中
- **风险描述：** 日历界面设计复杂，可能缺少专业设计师
- **缓解策略：**
  - 使用成熟的日历组件库（如FullCalendar等）
  - 参考优秀的日历应用设计
  - 先实现功能，后优化视觉
  - MVP阶段可以接受"工作但不完美"的UI
- **最低可接受标准：** 功能清晰可用，交互流畅，即使视觉不够精美

**风险3：测试和质量保证资源不足**
- **风险等级：** 中
- **风险描述：** 可能没有专职测试人员
- **缓解策略：**
  - 开发工程师进行自测
  - 个人使用即是最好的测试（dogfooding）
  - 关注核心功能的测试覆盖
  - 发布前在两个平台上完整测试主要流程
- **最低可接受标准：** 核心功能稳定，数据不丢失，算法准确

### MVP简化决策原则

如果在开发过程中需要进一步简化范围，按照以下优先级保留功能：

**P0 - 绝对核心（不可削减）：**
1. ✅ 知识点记录和管理（问题形式记录）
2. ✅ 基于艾宾浩斯的复习算法
3. ✅ 人性化自评系统（😟 🤔 😐 😊 🎯）
4. ✅ 复习提醒和任务列表
5. ✅ 基础日历视图（至少能看到每天的记录）
6. ✅ 本地数据存储（SQLite）

**没有以上功能，产品失去核心价值。**

**P1 - 高优先级（尽量保留，MVP必需）：**
7. 🔶 全局和单点复习频率系数调整
8. 🔶 日历详细视图和交互
9. 🔶 基础统计（今日统计：记录X个，复习Y个）
10. 🔶 知识点搜索和分类/标签
11. 🔶 每个知识点显示"下次复习时间"
12. 🔶 复习历史和评分趋势

**如果时间紧张，可部分简化，但会影响用户体验。**

**P2 - 中优先级（可延后到1.1版本）：**
13. 📌 日记功能
14. 📌 提醒事项功能
15. 📌 数据导出/导入
16. 📌 详细统计图表（每周/每月）
17. 📌 记忆标准自定义
18. 📌 长期抽查机制

**这些功能增强产品价值，但不影响核心验证。**

**P3 - 低优先级（1.1或更晚版本）：**
19. 🔹 记忆进度预测可视化（精美进度条）
20. 🔹 高级搜索和筛选（多条件组合）
21. 🔹 知识点批量操作
22. 🔹 主题和个性化设置
23. 🔹 全局快捷键
24. 🔹 系统托盘集成
25. 🔹 开机自启动

**这些是锦上添花的功能，MVP不需要。**

**决策流程：**

```
时间充足 (3-4个月)
  └─> 完整MVP：P0 + P1 + P2部分功能

时间一般 (2-3个月)
  └─> 标准MVP：P0 + P1
  
时间紧张 (1.5-2个月)
  └─> 最小MVP：P0 + P1部分功能（7-10）

极度紧张 (<1.5个月)
  └─> 超最小MVP：仅P0
      注意：此时需重新评估是否值得继续
```

**关键原则：**
- ✅ 始终保持P0功能完整和稳定
- ✅ 功能少但质量高，好过功能多但不稳定
- ✅ 能够验证核心假设（复习算法有效性）是第一优先级
- ✅ 每个降级方案都要确保产品仍然有价值

### 范围控制与变更管理

**范围蔓延防范：**
- 严格按照MVP功能列表开发，新需求记录到"未来版本"
- 每个新功能想法都要评估：这是否是MVP验证核心价值所必需的？
- 定期回顾进度，如果发现偏离MVP范围，及时调整

**变更决策标准：**
- 功能增加：只有当发现"没有它MVP无法验证"时才增加
- 功能删除：如果开发中发现复杂度超预期，按P0-P3优先级删减
- 功能调整：如果发现更简单的实现方式能达到同样目标，可以调整

**迭代策略：**
- MVP 1.0：核心价值验证版本（P0 + P1）
- MVP 1.1：功能完善版本（增加P2部分功能）
- MVP 1.2：体验优化版本（UI/UX打磨，性能优化）
- 2.0：阶段2（跨设备同步）
- 3.0：阶段3（移动端应用）

通过明确的范围界定、风险识别和优先级排序，MindReminder项目将以务实的MVP策略开始，逐步演进为功能完善的跨平台个人知识管理平台。

## 非功能需求

### 性能（Performance）

**NFR-P1: 应用启动性能**
- 冷启动时间 ≤ 3秒（首次打开应用）
- 热启动时间 ≤ 1秒（从系统托盘恢复）
- 目标平台：Windows 10+, macOS 10.14+

**NFR-P2: 界面响应性能**
- 用户操作响应时间 ≤ 200毫秒（按钮点击、输入等）
- 日历月份切换 ≤ 1秒
- 搜索结果响应时间 ≤ 500毫秒
- 界面渲染保持 ≥ 60 FPS

**NFR-P3: 数据处理性能**
- 支持100+知识点时界面仍然流畅
- 支持500+知识点时主要功能响应时间增长 ≤ 50%
- 支持1000+知识点时日历切换 ≤ 2秒
- 数据库查询响应时间 ≤ 100毫秒（典型场景）

**NFR-P4: 资源占用**
- 运行时内存占用 ≤ 300MB（正常使用场景，100个知识点）
- CPU占用 ≤ 5%（空闲时），≤ 20%（活跃使用时）
- 安装包体积 ≤ 150MB

### 可靠性（Reliability）

**NFR-R1: 数据完整性**
- 系统崩溃或异常退出后，数据不丢失
- 使用SQLite事务保证数据一致性
- 应用启动时自动验证数据库完整性
- 数据导入导出过程中进行格式和完整性验证

**NFR-R2: 自动备份**
- 每日自动备份数据库到本地备份文件夹
- 保留最近7天的备份文件
- 用户可手动触发备份
- 备份文件命名规范：`backup_YYYY-MM-DD.sqlite`

**NFR-R3: 应用稳定性**
- 应用崩溃率 ≤ 0.1%（每1000次使用会话）
- 关键功能（记录、复习、查看）可用性 ≥ 99.5%
- 异常退出后下次启动能恢复未保存数据

**NFR-R4: 算法准确性**
- 复习时间计算100%符合艾宾浩斯遗忘曲线算法规范
- 基于人性化评分的间隔调整准确执行
- 记忆标准判断准确触发
- 长期抽查机制按设定规则正常运行

### 隐私与安全（Privacy & Security）

**NFR-S1: 数据隐私**
- 所有数据存储在用户本地设备
- 不收集任何用户数据或使用统计
- MVP阶段应用无需联网，无网络请求
- 用户对数据拥有100%掌控权

**NFR-S2: 数据存储位置**
- Windows：`%APPDATA%/MindReminder/`
- macOS：`~/Library/Application Support/MindReminder/`
- 用户可访问和管理数据文件

**NFR-S3: 未来云同步安全（阶段2准备）**
- 数据模型设计包含加密预留字段
- 为端到端加密预留架构空间
- 云同步功能作为可选项，用户可选择仅本地存储

**NFR-S4: 应用权限**
- 仅请求必要的系统权限（文件读写、通知）
- 明确告知用户权限用途
- 不请求网络权限（MVP阶段）

### 可用性（Usability）

**NFR-U1: 易学性**
- 新用户无需教程能在5分钟内记录第一个知识点
- 核心功能一目了然，操作直观
- 提供可选的"快速开始指南"（用户可跳过）
- 界面元素清晰明确，避免歧义

**NFR-U2: 操作效率**
- 快速记录知识点 ≤ 3次点击
- 完成一次复习评分 ≤ 2次点击
- 支持键盘快捷键加速操作
- 常用功能在主界面可直接访问

**NFR-U3: 反馈及时性**
- 所有用户操作提供即时视觉反馈（≤ 100毫秒）
